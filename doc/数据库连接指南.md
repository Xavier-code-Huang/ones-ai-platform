## 3. 连接方式

### 方式一：服务器上用 psql（推荐）

```bash
# 先 SSH 到服务器，然后：
sudo docker exec -it onesai-backend psql -U onesai -d ones_ai_platform -h 127.0.0.1 -p 9622
```

进入 psql 后可直接执行 SQL：
```sql
-- 查看所有表
\dt

-- 查看表结构
\d users
\d tasks

-- 退出
\q
```

### 方式二：服务器上直接执行 SQL 命令

```bash
# 不进入交互模式，直接执行一条 SQL
sudo docker exec onesai-backend psql -U onesai -d ones_ai_platform -h 127.0.0.1 -p 9622 -c "SELECT * FROM users;"
```

### 方式三：本地远程连接（需网络可达）

如果你的电脑能访问 `172.60.1.35:9622`，可以用任意数据库客户端直接连：

## 4. 数据库表结构（12 张表）

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `users` | 用户 | id, ones_email, display_name, role |
| `servers` | 远程服务器 | id, name, host, ssh_port |
| `server_credentials` | SSH 凭证 | server_id, username, encrypted_password |
| `tasks` | 任务 | id, user_id, server_id, status, submitted_at |
| `task_tickets` | 工单处理结果 | task_id, ticket_id, status, result_conclusion, result_report |
| `task_logs` | 执行日志 | task_id, content, log_type |
| `evaluations` | 人工评价 | task_ticket_id, passed, reason |
| `configs` | 系统配置 | key, value |
| `user_daily_stats` | 每日统计 | user_id, date, task_count |
| `external_teams` | 外部团队 | team_name, api_key, is_active |
| `external_team_members` | 外部成员 | team_id, member_name |
| `external_logs` | 外部日志 | team_id, member_name, status, duration |

## 5. 常用查询示例

```sql
-- 查看所有用户
SELECT id, ones_email, display_name, role, created_at FROM users ORDER BY id;

-- 查看最近10个任务
SELECT t.id, u.display_name, t.status, t.ticket_count, t.submitted_at
FROM tasks t JOIN users u ON t.user_id = u.id
ORDER BY t.submitted_at DESC LIMIT 10;

-- 查看工单处理结果
SELECT ticket_id, status, result_conclusion, duration
FROM task_tickets
WHERE task_id = 1
ORDER BY id;

-- 统计每个用户的任务数
SELECT u.display_name, COUNT(t.id) as task_count
FROM users u LEFT JOIN tasks t ON u.id = t.user_id
GROUP BY u.display_name ORDER BY task_count DESC;

-- 查看外部团队
SELECT team_name, is_active, created_at FROM external_teams;

-- 导出数据为 CSV（在 psql 中）
\copy (SELECT * FROM task_tickets) TO '/tmp/tickets.csv' CSV HEADER;
```

## 6. 数据库备份与恢复

```bash
# 备份
sudo docker exec onesai-backend pg_dump -U onesai -d ones_ai_platform \
  -h 127.0.0.1 -p 9622 -f /tmp/db_backup.sql
sudo docker cp onesai-backend:/tmp/db_backup.sql /tmp/db_backup.sql

# 恢复（慎用！会覆盖现有数据）
sudo docker cp /tmp/db_backup.sql onesai-backend:/tmp/db_backup.sql
sudo docker exec onesai-backend psql -U onesai -d ones_ai_platform \
  -h 127.0.0.1 -p 9622 -f /tmp/db_backup.sql
```

## 8. 注意事项

1. **只读操作即可**，请勿随意 INSERT/UPDATE/DELETE 生产数据
2. 如需修改数据，请先备份
3. 数据库密码为明文存储在此文档中，请注意保密
4. 如果 psql 提示 `connection refused`，检查容器是否运行：`sudo docker ps | grep onesai`
