# AI 分析过程持久化存储 + 历史回放

## 背景

当前 "观测容器" 按钮通过 WebSocket → SSH → `docker logs -f` 实时查看容器输出。但容器退出后日志就不可用了，用户无法回看 AI 分析过程。

## 现有基础

经调研，**基础设施已经很完善**：

| 组件 | 现状 | 说明 |
|------|------|------|
| `_save_log()` | ✅ 已实现 | 普通 stdout 行已写入 `task_logs` 表 |
| `task_logs` 表 | ✅ 已存在 | 有 `task_id, task_ticket_id, log_type, content, phase_name, timestamp` |
| `/api/tasks/{id}/logs` | ✅ 已实现 | 能返回历史日志 |
| WebTerminal.vue | ✅ 已实现 | xterm.js 终端组件 |

**关键缺口**：
1. 容器终端的输出没有直接存入数据库 — 但 stream-json 的人类可读翻译行已经在 stdout 中被 `_save_log` 保存
2. 前端没有「历史回放」模式 — WebTerminal 只能连接活着的容器

## 方案

改动极小，只需要在前端加一个回放模式即可。

### 后端
- task_executor.py 无需修改（stdout 已持久化）
- tasks.py 新增工单级日志 API

### 前端
- TaskDetailView.vue 按钮文案切换：执行中 → "观测容器"；结束 → "📼 历史回放"
- WebTerminal.vue 新增 replay 模式，从 API 加载历史日志并用 xterm 渲染
