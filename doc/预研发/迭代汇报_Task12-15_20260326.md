## 一、本次迭代概要

本次迭代完成了任务执行流水线重构的 **Task 12 至 Task 15**，涵盖前端交互升级、系统稳定性强化和多 Agent 串行管线架构落地三大模块。所有代码已通过前端生产构建验证（2242 模块，17.3s）。

## 二、核心交付物

### 2.1 任务提交页全面重构（Task 12 + 13.3）

- **现代化卡片式布局**：参照 GitHub Actions dispatch UI 重新设计，分为"执行环境"和"工单列表"两个独立区块
- **CodePathSelect 集成**：代码目录输入支持历史路径下拉选择，按使用频率排序，支持一键删除
- **拖拽排序**：工单卡片支持 HTML5 原生拖拽调整执行顺序
- **AI 预分析按钮**：手动触发模式，调用后端 GLM-5 模型生成推荐提示词，用户可编辑后确认
- **多工单号自动拆分**：输入包含逗号/空格分隔的多个工单号时自动拆分为独立行

### 2.2 工单卡死防护机制（Task 14）

- **工单级去重**：同一任务内禁止提交重复工单号，从源头消除重复执行
- **心跳超时机制**：任何工单 stdout 无输出超过 20 分钟自动判定卡死，强制终止并标记失败
- **全局 Watchdog 守护**：独立后台协程，每 5 分钟扫描数据库，清理 running 超过 3 小时的工单和 4 小时的任务
- **启动自愈**：系统重启时自动清理历史遗留的孤立 running 状态工单

### 2.3 Claude CLI Subagent 管线（Task 15）

- **4 个 Subagent 定义文件**：分析(analyzer)→修改(modifier)→验证(verifier)→报告(reporter) 串行管线
- **Runner 管线集成**：默认 prompt 包含 subagent 管线指令，每阶段输出 `[PHASE]` 标记供后端推进时间线
- **agent-teams 规范注入**：用户自定义的 agent-teams 目录下的 .md 文件自动载入并注入到每个 subagent prompt
- **max-turns 扩充**：从 50 提升至 100，确保多 subagent 调用不受轮数限制
