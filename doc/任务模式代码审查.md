[tasks.py:60-73](file:///F:/Ones-AI专项开发/ones-ai-platform/backend/tasks.py#L60-L73)

```python
class TaskInfo(BaseModel):
    id: int
    user_id: int
```

`task_mode` 写入了数据库但**没有在响应模型中返回**。前端提交后跳转到任务详情页时，无法知道这个任务是什么模式。

## 🟡 建议-1: `git diff` 在 Docker 容器内执行，`code_dir` 路径上下文需注意

[ones_task_runner.py:868-881](file:///F:/Claude/lango-claude-deploy/ones_task_runner/ones_task_runner.py#L868-L881)

```python
if task_mode in ("fix", "auto") and code_dir:
    diff_proc = subprocess.run(
        ["git", "diff"],
        cwd=code_dir,
        ...
    )
```

Runner 运行在 Docker 容器内，`code_dir` 是通过 `-v` 挂载进来的宿主机路径。`git diff` 能正常工作的前提是：
1. `code_dir` 是 git 仓库（有 `.git` 目录）
2. Docker 容器中 git 已安装（ones-AI 脚本中有 `git config --global` 所以 git 肯定存在）

**风险等级**：低。AOSP 代码目录通常都是 git repo。但若 `code_dir` 指向 repo-tool 管理的目录（多个子 git），`git diff` 只能输出当前目录层级的 diff，不包含子仓库。

**建议**：可以考虑用 `repo diff` 或 `find . -name .git -exec git -C {} diff \;` 做更全面的收集。暂不需要改，记录为后续优化点。

## 🟡 建议-2: `execute-task.md` 模板里的"保守方案"与 `fix` 模式 prompt 可能矛盾

[execute-task.md:37](file:///F:/Claude/lango-claude-deploy/ones_task_runner/commands/execute-task.md#L37)

```markdown
- 对于不确定的改动，优先采用保守方案
```

这行告诉 AI 不确定时要保守。而 `fix` 模式的 prompt（注入在最后）说"**禁止只输出分析报告**"。

由于模式 prompt 放在 skill template **之后**（更靠近 prompt 末尾），Claude 通常会优先遵循后面的指令。所以实际上不冲突，但语义上有张力。

**建议**：下次更新 `execute-task.md` 时可以删除这行"保守方案"的描述，让模式 prompt 完全决定行为。暂不需要改。
