## 1. 项目概述

### 1.1 背景

ones-AI 是 lango-AI 生态中的工单自动化处理工具，当前仅有命令行入口和简陋的监控页面。
需要构建一个 **Web 化的全功能平台**，让用户通过浏览器提交工单任务、实时查看处理过程、
接收企业微信通知，同时为管理层提供数据可视化的管理抓手。

### 1.2 核心价值

| 利益方 | 价值 |
|--------|------|
| **开发者** | 无需 SSH 登录服务器，浏览器提交工单即可自动处理 |
| **管理层** | 清晰掌握 AI 工具使用情况、投入产出比、团队采纳率 |
| **组织** | 降低工单处理时间，提升 AI 辅助开发的可衡量效果 |

### 1.3 技术约束

- 部署在 `172.60.1.35` 服务器（Ubuntu），独立端口
- 复用 `lango-remote` 的 auth / SSH / DB 架构模式
- 数据库使用 PostgreSQL（200 人并发规模）
- 前端使用 Vue 3 + Vite
- 后端使用 Python FastAPI
- 目标服务器已预装 `ones_task_runner.py`
- **AI 模型**: Claude (via GLM-5 代理) — `ANTHROPIC_BASE_URL=https://open.bigmodel.cn/api/anthropic`
- **外部服务**: ONES API 网关、Gerrit (3 个实例)、企业微信 — 地址均通过管理页面配置

## 3. 功能需求

### FR-001 用户认证 — ONES 账号登录

**描述**: 用户使用 ONES 系统账号（即企业微信邮箱 + 密码）登录平台。

**验收标准**:
1. 登录页面提供邮箱和密码输入框
2. 提交后，后端使用 ONES API 验证账号密码有效性（模拟 ONES MCP 的认证方式）
3. 验证成功后生成 JWT Token，返回用户信息
4. 验证失败返回明确的错误提示（账号不存在 / 密码错误）
5. Token 有效期可配置，过期需重新登录
6. 登录状态持久化（localStorage），刷新页面不丢失

**关联**: 设计文档 §4.1 | lango-AI: `Auth-Claude.sh`

---

### FR-002 服务器列表展示

**描述**: 用户登录后展示所有可用服务器列表。

**验收标准**:
1. 展示系统中注册的所有服务器（名称、IP、状态）
2. 服务器状态实时标识（在线/离线/未知）
3. 用户可以看到所有服务器，但仅能操作已验证凭证的服务器
4. 已绑定凭证的服务器显示已验证标识
5. 未绑定凭证的服务器显示"需要验证"标识

**关联**: 设计文档 §4.3 | lango-remote: `machines.py`, `ssh_pool.py`

---

### FR-003 服务器凭证绑定

**描述**: 用户首次使用某台服务器时，需输入该服务器的 SSH 账号密码进行校验。

**验收标准**:
1. 点击未验证的服务器弹出凭证输入对话框（用户名 + 密码）
2. 一个用户在同一服务器上可绑定多组凭证（多个账号）
3. 后端通过 SSH 实际连接目标服务器进行验证
4. 验证成功：弹出"登录成功"提示，凭证加密存储
5. 验证失败：弹出警告"账号密码错误"或"服务器上没有此用户"
6. 已绑定的凭证可管理（查看/删除/重新验证）
7. 密码使用 AES 加密存储，非明文

**关联**: 设计文档 §4.3, §4.2 | lango-remote: `crypto.py`

---

### FR-004 任务创建

**描述**: 用户在已验证的服务器上创建工单处理任务。

**验收标准**:
1. 任务创建表单支持输入多个工单号（可逐个添加或批量粘贴）
2. 每个工单号可附带：
   - 补充说明（文本，**可选**）
   - 代码目录位置（文本，可选）
   - 编译命令（文本，标记为"暂未开放"灰显不可编辑）
   - 是否执行编译测试（开关，标记为"暂未开放"灰显不可操作）
3. 工单号支持自动格式化（纯数字自动补 `ONES-` 前缀）
4. 支持逗号/空格/换行分隔的批量输入
5. 提交前显示任务预览确认
6. 选择目标服务器上的账号（如果有多个凭证）

**关联**: 设计文档 §4.4 | `ones_task_runner.py`: `normalize_ticket_id()`

---

### FR-005 任务执行与 SSH 转发

**描述**: 用户确认任务后，平台通过 SSH 将任务转发到目标服务器执行 ones-AI。

**验收标准**:
1. 任务提交后进入执行队列
2. 平台通过 SSH 连接到目标服务器
3. 在目标服务器上调用 `ones_task_runner.py` 执行工单任务
4. 传递完整的任务参数（工单号、补充说明等）
5. 执行使用用户绑定的服务器凭证
6. 同一服务器的任务串行执行（避免资源冲突）
7. 任务超时机制（可配置，默认 3600s/工单）

**关联**: 设计文档 §4.5 | lango-remote: `ssh_pool.py` | `ones_task_runner.py`: `ClaudeExecutor`

### FR-014 外部服务配置管理 [Phase 1]

**描述**: 管理员可在管理页面配置外部服务地址和凭证，无需修改代码或重启服务。

**验收标准**:
1. 管理页面提供配置表单，包含以下外部服务：
   - **ONES API 网关**: 地址 `http://172.60.1.103:10076`（业务数据接口，v3 路径）
   - **ONES 直连地址**: `http://10.10.10.113:30011`（登录认证专用，网关不提供登录）
   - **Gerrit 实例**: 3 组（每组含地址、HTTP Token、账号、密码）
   - **企业微信**: Corp ID、Agent ID、Secret、Webhook URL
2. 配置项加密存储（凭证类字段 AES 加密）
3. 配置变更即时生效，无需重启
4. 提供"测试连接"按钮验证各服务连通性
5. Gerrit 配置一期仅展示存储，不实际调用

**关联**: 设计文档 §4.10

### FR-015 AI 模型与提示词设计 [Phase 1]

**描述**: ones-AI 使用 Claude (通过 GLM-5 代理) 处理工单，需优化提示词设计以提升准确率。

**验收标准**:
1. ones_task_runner 的 AI 调用配置集中管理（模型、API 地址、超时等）
2. 针对大型代码仓库（400G AOSP）设计分层分析策略提示词：
   - 第一步：分析工单描述，提取关键信息
   - 第二步：根据关键词定位相关模块/目录
   - 第三步：聚焦核心代码进行修改
3. runner 中的提示词使用占位符配置，便于后续调优
4. AI 模型配置：
   - `ANTHROPIC_BASE_URL` = `https://open.bigmodel.cn/api/anthropic`
   - `ANTHROPIC_DEFAULT_SONNET_MODEL` = `glm-4.7`
   - `ANTHROPIC_DEFAULT_OPUS_MODEL` = `glm-5`

**关联**: 设计文档 §4.11 | `ones-AI` 脚本环境变量

### FR-017 ones_task_runner 改造 [Phase 1]

**描述**: 增强 runner 以支持 Web 平台调用、批量参数、结构化输出。

**验收标准**:
1. 支持 `--json-output` 模式（每处理完一个工单输出一行 JSON）
2. 支持进度上报标记 `[PROGRESS]`（便于 Web 端解析更新进度条）
3. 补充说明（`--notes`）参数改为可选，无说明时不报错
4. 支持批量传入工单号 + 对应描述（描述为可选）
5. Gerrit 配置占位符写入 config.yaml（`__GERRIT_HOST_1__` / `__GERRIT_TOKEN_1__` 等）
6. **不影响**现有 CLI 模式的运行行为

**关联**: 设计文档 §7 | `ones_task_runner.py`: `ClaudeExecutor`, `parse_args()`

## 4. 非功能需求

### NFR-001 安全性
1. 所有密码使用 bcrypt 哈希存储（平台密码）或 AES 加密存储（SSH/Gerrit 凭证）
2. JWT Token 使用 HS256 签名，密钥从环境变量读取
3. API 端点全部需 Bearer Token 认证（除登录接口）
4. 管理页面需 admin 角色权限
5. SSH 凭证解密密钥不存储在数据库中

### NFR-002 性能
1. 页面首屏加载 ≤ 2 秒
2. API 响应 ≤ 500ms（统计查询 ≤ 2s）
3. WebSocket 日志延迟 ≤ 1 秒
4. 支持 200 并发用户

### NFR-003 可用性
1. 响应式布局（最小支持 1280px）
2. 暗色主题，视觉高级
3. 所有操作有加载状态指示
4. 错误信息友好

### NFR-004 可维护性
1. 后端 API 文档自动生成（Swagger）
2. 配置项集中管理，环境变量覆盖
3. 日志分级，审计日志

## 6. 数据需求

### 6.1 需要持久化的数据
- 用户信息、服务器凭证、任务记录、工单明细、执行日志
- 用户评价、企微通知记录
- **外部服务配置**（ONES/Gerrit/企微地址和凭证）

### 6.2 数据保留策略
- 任务记录和执行结果：永久保留
- 执行日志原文：保留 90 天（可配置）
- 统计数据：永久保留
