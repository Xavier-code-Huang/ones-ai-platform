# ones-AI 外部团队日志上报接口文档

> **版本**: v1.0.0 &nbsp;|&nbsp; **更新日期**: 2026-03-26 &nbsp;|&nbsp; **维护方**: ones-AI 平台组

---

## 1. 概述

ones-AI 平台开放日志上报接口，允许外部团队将自研工单处理系统的使用数据汇入 ones-AI 统一监控平台。平台将自动统计和展示各团队的使用次数、成员明细、成功率、趋势等。

**接入流程：**

1. 联系 ones-AI 管理员，提供团队名称 → 获取 **API Key**
2. 在你的系统中集成上报接口（见下方说明）
3. 在 ones-AI 管理后台即可查看本团队的统计数据

---

## 2. 基础信息

| 项目 | 值 |
|------|------|
| **接口地址** | `http://172.60.1.35:9621/api/external/report` |
| **请求方法** | `POST` |
| **认证方式** | HTTP Header：`X-API-Key: <你的API Key>` |
| **Content-Type** | `application/json` |
| **频率限制** | 单次上报不超过 **100 条**日志 |

---

## 3. 请求格式

### 3.1 Headers

```
Content-Type: application/json
X-API-Key: <管理员分配的64位API Key>
```

### 3.2 请求体（JSON）

```json
{
  "logs": [
    {
      "member_name": "张三",
      "ticket_id": "ONES-123456",
      "action_type": "process",
      "status": "completed",
      "duration": 120.5,
      "summary": "修复了空指针问题"
    },
    {
      "member_name": "李四",
      "ticket_id": "ONES-789012",
      "action_type": "analysis",
      "status": "failed",
      "duration": 45.0,
      "summary": "无法复现该问题"
    }
  ]
}
```

### 3.3 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `logs` | Array | ✅ | — | 日志条目数组，1~100 条 |
| `logs[].member_name` | String | ✅ | — | 操作人姓名（首次上报会自动注册为团队成员） |
| `logs[].ticket_id` | String | ❌ | `""` | 关联的工单编号 |
| `logs[].action_type` | String | ❌ | `"process"` | 操作类型，建议值：`process`（处理）、`analysis`（分析）、`fix`（修复） |
| `logs[].status` | String | ❌ | `"completed"` | 处理结果：`completed`（成功）或 `failed`（失败） |
| `logs[].duration` | Number | ❌ | `0` | 处理耗时（秒） |
| `logs[].summary` | String | ❌ | `""` | 处理摘要/备注 |

---

## 4. 响应格式

### 4.1 成功响应（200）

```json
{
  "success": true,
  "inserted": 2,
  "team": "视觉质量团队"
}
```

| 字段 | 说明 |
|------|------|
| `success` | 是否成功 |
| `inserted` | 本次实际插入的日志条数 |
| `team` | 团队名称（用于确认身份） |

### 4.2 错误响应

| HTTP 状态码 | 含义 | 示例 |
|-------------|------|------|
| `400` | 参数错误 | `{"detail": "日志列表不能为空"}` |
| `403` | 认证失败 | `{"detail": "无效的 API Key 或团队已禁用"}` |
| `422` | 格式错误 | 缺少必填字段或类型不匹配 |

## 5. 代码示例

### Python

```python
import requests

API_URL = "http://172.60.1.35:9621/api/external/report"
API_KEY = "你的API Key"

response = requests.post(API_URL, json={
    "logs": [
        {
            "member_name": "张三",
            "ticket_id": "ONES-123456",
            "action_type": "process",
            "status": "completed",
            "duration": 120.5,
            "summary": "修复了空指针问题"
        }
    ]
}, headers={"X-API-Key": API_KEY})

print(response.json())
```

## 6. 最佳实践

1. **批量上报**：建议攒一批日志一起上报（如每 5 分钟或每 10 条），减少请求次数
2. **member_name 保持一致**：同一人的 `member_name` 请保持一致，平台按此字段统计个人使用量
3. **duration 精确到秒**：填写从接到工单到处理完成的实际耗时
4. **status 如实填写**：`completed` 代表成功处理，`failed` 代表处理失败，平台会统计成功率
