# Web Terminal 容器接入交互设计（重构版）

基于"顺手、符合逻辑、合理且美观优雅、完全自动化"的要求，我们将摒弃之前构思的浮动多窗口或底部Tab栏方案（这些方案往往显得杂乱且操作繁琐），采用**无感集成**和**状态联动**的现代化交互设计。

## 🎨 核心交互理念：所见即所得，自动跟随

1. **终端内嵌，不弹窗、不浮动**
   - 终端直接嵌入在右侧**"工单详情"**面板中（位于阶段时间线与处理详情之间）。
   - 用户点击左侧工单卡片切换时，右侧终端**自动切换**为对应工单的终端会话。
   - 极其顺手：看哪个工单的结论，就顺便能看到哪个工单的实时终端。

2. **状态驱动的完全自动化**
   - **执行中（Running）**：左侧选中时，右侧终端**自动展开并连接**，实时刷日志，无需手动点击。
   - **排队中（Pending）**：隐藏终端。
   - **已完成/失败（Completed/Failed）**：显示"容器已停止"状态和一个优雅的 🚀 **重新唤醒容器**按钮。
   - 点击唤醒后：**自动**发API启动容器 → **自动**连接终端 → **自动**输入 `/resume` → 终端展开准备接管。整个过程一气呵成。

3. **优雅的视觉过渡**
   - 终端面板具有丝滑的展开/折叠动画。
   - 终端本身采用美观的深色微光边框设计（仿高级IDE）。
   - 切换工单时，终端内容瞬间清空并重建连接，绝无残留。

## Proposed Changes

### 1. 前端 — 视图级整合 (TaskDetailView.vue)

- **移除全局的终端按钮和顶级面板**。
- 将 `<WebTerminal>` 组件移动到 `.ticket-result-card`（右侧工单详情卡片）的顶部或时间线下方。
- `<WebTerminal>` 与当前选中的 `selectedTicket.id` 强绑定。
- 增加终端面板的显隐逻辑：
  ```
  如果 selectedTicket.status === 'running'
    → 自动显示终端并连接
  如果 selectedTicket.status === 'completed' or 'failed'
    → 默认折叠，提供"唤醒容器进行干预"操作区。唤醒后自动展开。
  ```

### 2. 前端 — 组件升级 (WebTerminal.vue)

- 支持传入 `ticketDbId`、`status` 等 Props。
- 优化尺寸适应：利用 `ResizeObserver` 配合 xterm 的 FitAddon，确保在页面布局发生微调时终端不闪烁。
- 增加**内置自动化工作流**支持：如果在唤醒状态且收到连接成功事件，可触发 `/resume` 输入。

### 3. 后端 — 更细粒度的控制 (terminal_ws.py)

- 接口仍为 `/ws/tickets/{ticket_db_id}/terminal`。
- 新增 `auto_resume` 参数：支持建立连接时自动向 PTY 发送 `/resume\n` 命令，省去用户手动输入。

### 4. 后端 — 重启与状态 API (tasks.py)

- `GET /api/tasks/{task_id}/tickets/{ticket_db_id}/container`（检查状态）
- `POST /api/tasks/{task_id}/tickets/{ticket_db_id}/container/start`
  - 如果容器挂载了某些特殊卷需要调整，在 start 前进行。目前直接 `docker start {container_name}` 即可。

## User Review Required

> [!TIP]
> **全自动化的"唤醒并恢复"交互流**：
> 用户点击已完成工单的**"唤醒干预"**按钮 → 界面 Loading并显示动画 → 后端静默 `docker start` 容器 → 前端 WebSocket 连接 → 后端注入 `/resume` → 终端展开，显示 AI 回答的恢复过程。
> **您同意这种内嵌工单详情卡片内的展示方式和全自动化流程吗？** 这比满屏飘的浮动窗口更好管理且直观！

## Verification Plan

### 手动验证

1. **执行中场景测试**：
   - 任务处于执行状态时，点击不同工单卡片。
   - 期望：右侧详情随之切换，内部终端瞬间同步切换并重连至对应的运行中容器。
2. **干预场景测试**：
   - 选中一个已完成的工单。
   - 点击"唤醒容器干预"按钮。
   - 期望：按钮转为 Loading，随即终端面板优雅展开，显示 `[系统] 正在唤醒容器...`，然后直接看到 AI 恢复 `/resume` 的输出，用户可立即开始输入。键盘输入流畅稳定。
3. **环境保留测试**：
   - 刷新页面，保持刚刚干预的工单被选中，终端应能自动回连并保持之前的上下文。
