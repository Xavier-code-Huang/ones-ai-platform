## 工单 4：Bug 修复 — Argument list too long 错误

**标题**: 修复 Runner 执行时 "Argument list too long" 致命错误

**描述**: 当 Agent Teams 目录包含大量 .md 文件时，拼接后的 prompt 超过 Linux ARG_MAX 限制（~2MB），导致 claude CLI 启动失败。需将 prompt 从命令行参数改为 stdin 管道传入。

**详情**:
- 原逻辑: `subprocess.run(command, shell=True)` 将 prompt 拼入 `-p` 参数
- 新逻辑: `subprocess.Popen(args, stdin=PIPE)` + `proc.communicate(input=prompt.encode('utf-8'))`
- 命令参数中 `-p` 后跟 `-`（表示从 stdin 读取）
- 超时处理新增 `proc.kill()` + `proc.communicate()` 回收子进程
- 修复所有 `result.stdout`/`result.returncode` 引用为 `stdout_bytes.decode()`/`proc.returncode`
- 涉及文件: `ones_task_runner.py`

**预估工时**: 3 小时

## 工单 5：Patch 回收功能

**标题**: Runner 执行后自动收集 git diff 作为 patch 回传

**描述**: 在 fix 和 auto 模式下，任务执行完成后自动在代码目录运行 `git diff`，将差异内容存入执行结果中回传给平台。

**详情**:
- run() 方法中，execute() 返回后检查 task_mode 是否为 fix/auto
- 若代码目录存在，执行 `subprocess.run(["git", "diff"], cwd=code_dir, timeout=30)`
- 收集的 diff 内容存入 `result['patch']` 字段
- analysis 模式下跳过收集
- 异常情况静默处理（logger.debug），不影响主流程
- 涉及文件: `ones_task_runner.py`

**预估工时**: 2 小时

## 工单 6：前端首次加载性能优化

**标题**: ones-AI 平台前端首次加载性能优化

**描述**: 用户反馈首次打开平台页面加载缓慢。通过 Vite 分包配置和 Nginx 压缩缓存策略优化首屏加载速度。

**详情**:
- vite.config.js 新增 build.rollupOptions.output.manualChunks 配置
  - Element Plus + Icons 单独打包为 `element-plus` chunk
  - Vue + vue-router + pinia 打包为 `vue-vendor` chunk
- nginx.conf 新增 gzip 压缩（level 5, 1024 最小阈值）
- nginx.conf 新增 /assets/ 路径 1 年长期缓存（Vite 文件名含 hash）
- nginx.conf 对 index.html 设置 no-cache（确保更新即时生效）
- 预期效果: JS 包体积减少 60-80%，二次访问零网络传输
- 涉及文件: `vite.config.js`, `nginx.conf`

**预估工时**: 3 小时
